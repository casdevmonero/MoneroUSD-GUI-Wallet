Import wallet from seed phrase

Restore a wallet from your 25-word seed. Data is sent only to your local Monero USD wallet RPC and is never stored or logged. Start the wallet RPC with --wallet-dir=/path/to/dir (no --wallet-file) so it can create the restored wallet. If you see "Wallet RPC unreachable" or "Address already in use", ensure the wallet RPC is running and Settings → Wallet RPC URL is correct. If port 27750 is in use, run HAVEN_WALLET_RPC_PORT=27751 ./start-wallet-rpc.sh and update the URL accordingly.

Use the same language shown by CLI command seed_language.

Wallet RPC connection

Start the RPC with --wallet-file=YourWallet or with --wallet-dir=/path/to/dir for Import from seed. In browser mode the app proxies to the Wallet RPC URL (localhost only). If 27750 is in use, use --rpc-bind-port=27751 and update the URL accordingly.

USDm uses a local node only. Set Daemon URL to the same node your CLI wallet uses (default port 17750). The app connects the wallet RPC directly to this node via set_daemon. Run havend from haven-main, then start the wallet RPC. If balance is wrong after sync, use Dashboard → Rescan to rebuild the wallet view (no local SQLite; wallet state is in the RPC).
